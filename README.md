# AltMart Server

Лёгкий self-hosted сервер для каталога Android-приложений (форк протокола oldmarket).
FastAPI + aiosqlite, без SQLAlchemy/Flask — минимум зависимостей.

## Установка

```bash
pip install -r requirements.txt
# опционально, для tui.py и live_scrapper.py:
pip install -r requirements-tools.txt
```

## Первый запуск

```bash
python3 scripts/init_db.py
```
Скрипт спросит логин/пароль и создаст структуру БД + твой premium-аккаунт.
**Пароль больше не хранится в коде** — вводится интерактивно и сразу хешируется (bcrypt).

Положи скачанные APK/иконки/скриншоты в `downloaded/apks`, `downloaded/html/...`
(структура папок видна в `main.py`, в `static_dirs`) — этот каталог в `.gitignore`,
он не должен попадать в git.

## Запуск сервера

```bash
python3 main.py
# или
uvicorn main:app --host 0.0.0.0 --port 5000
```

## Управление (CLI и TUI)

```bash
python3 cli.py db stats
python3 cli.py user create --name vitalokek2   # спросит пароль интерактивно
python3 tui.py                                  # графическая панель на rich
```

## Что изменено по сравнению с черновой версией

- Убран захардкоженный пароль из `cli.py`/`scripts/init_db.py` — теперь только интерактивный ввод + bcrypt-хеш.
- Пароли пользователей хранятся хешированными (`bcrypt`), а не в открытом виде.
- `/api/login` больше не ставит фейковую "as-if" куку — нормальная HttpOnly-кука с user_id.
- `/api/app/{id}/review` берёт реальный username из БД вместо захардкоженного.
- `/api/categories` использует параметризованный SQL-запрос вместо f-string.
- Добавлен `security.py` — лёгкая защита, перенесённая идейно из оригинального oldmarket-server:
  - блокировка IP при подборе пароля (5 неудач за 15 минут → бан на час);
  - лимит регистраций с одного IP (3 в сутки);
  - дедупликация скачиваний по IP (счётчик downloads не накручивается повторными запросами).
- Удалены из репозитория: дублирующая старая версия сервера (`server/` на чистом sqlite3),
  35+ МБ перехваченного HTTP-трафика (`requests/`), декомпилированный APK (`sourcecode/`),
  служебные одноразовые скрипты анализа (`peek.py`, `WebAnalytics/`) — они не нужны для
  работы сервера. `.db`-файлы и `__pycache__` теперь в `.gitignore`.
